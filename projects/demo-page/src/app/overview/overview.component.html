The ng-table-virtual-scroll package provides an opportunity to use Angular Material cdk-virtual-scroll-viewport with mat-table

Installation

To install package by npm:
$ npm install -save ng-table-virtual-scroll

Note: for angular version <= 12, please use library version 1.3.*

To make it work in your project:
1) Import module: 




  
    Note: you need to install and configure
    cdk-virtual-scroll-viewport
     (ScrollingModule) and
    mat-table  (MatTableModule) before.
    TableVirtualScroll only make them work together properly
  




  2) Use tvsItemSize directive on <cdk-virtual-scroll-viewport>




  3) Use new TableVirtualDataSource() as [dataSource] of a <table mat-table>




tvsItemSize directive




  When all rows of the table are the same fixed size, you can wrap your table[mat-table] with cdk-virtual-scroll-viewport
  container and add the tvsItemSize directive on it.





  To add come configuration, you can add additional properties:
  

  tvsItemSize - the row height in px (default: 48)
  

  headerHeight - the header row height in px (default: 56)
  

  footerHeight - the footer row height in px (default: 48)
  

  
    Note: tvsItemSize/headerHeight/footerHeight properties only required for calculations - you need to style your table separately by css
  




  headerEnabled - is the header row in the table (default: true)
  

  footerEnabled - is the footer row in the table (default: false)
  

  bufferMultiplier - the size of rendered buffer (default: 0.7). The 'bufferMultiplier * visibleRowsCount'
  number of rows will be rendered before and after visible part of the table





  Don't forget to set height on the cdk-virtual-scroll-viewport




VirtualTableDataSource




  The tvsItemSize requires the VirtualTableDataSource to be provide as mat-table[dataSource].
  It is the same as the MatTableDataSource, but with few adjustments which was made so the virtual scroll could work.
